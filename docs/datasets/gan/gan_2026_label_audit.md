# GAN 2026 Dataset Label Audit

Date: 2026-05-14

## Context

This audit covers the files in `data/raw/gan_2026/`. Three papers provide the methodological
foundation for the dataset and scoring:

- **Fang et al. (Epilepsia 2025)** — epilepsy information extraction from KCH clinic letters using
  LLMs (epilepsy type, seizure type, ASMs, associated symptoms). The broader extraction context;
  the data used here is the 280-letter annotated subset from King's College Hospital.

- **Holgate et al. (BioNLP 2025)** — defines the **seizure frequency measurement system**. Seizure
  frequency is expressed as a single numeric value: number of seizures per month. One or more per
  month = integer; less than one per month = decimal. Two special values: `1000` (seizures
  referenced but frequency unclear/unspecified) and `0` (no seizure information at all). Annotated
  1,480 KCH EHRs with this scheme; inter-annotator Cohen's kappa = 0.84 on 300 gold-standard EHRs.

- **Gan et al. (arXiv 2603.11407, 2026)** — **the paper that generated this dataset**. Defines the
  Label Scheme taxonomy used in the JSON labels, describes the GPT-5-based synthetic letter
  generation pipeline, and documents scoring methodology (Purist and Pragmatic evaluation methods).

---

## Dataset Overview

### `synthetic_data_subset_1500.json`

A subset of 1,500 records drawn from a corpus of **15,099 synthetic NHS-style neurology clinic
letters** generated by Gan et al. using GPT-5 as a teacher model. Source indices span 4–17297,
confirming this is a sample from the full ~17,000-record synthetic corpus.

Each letter was generated from one of 10 privacy-checked real KCH base letters combined with a
short seizure-frequency description. GPT-5 generated the full letter and also produced:
- A chain-of-thought analysis (`analysis`)
- A final Label Scheme label (`seizure_frequency_number[0]`)
- An evidence span from the letter (`seizure_frequency_number[1]`)

All 1,500 records passed the **Step 7 verification filter**: GPT-5's inferred label had to exactly
match the ground-truth label from the description-annotation step (Step 4). This is why
`labels_match_all_categories=True` for all records — it reflects the construction-time filter, not
a post-hoc quality check. It is correct by design.

The `reference` field is a **secondary cross-check pass** that re-derives the label independently
from the letter. It is not the gold label. `seizure_frequency_number[0]` is gold.

| Field | Value |
|---|---|
| Total records | 1500 |
| Source row index range | 4 – 17297 |
| Admin/non-clinical letters | 54 (3.6%) |
| Clinical letters | 1446 (96.4%) |
| Label-reference agreement (clinical) | 1249 / 1446 (86.4%) |
| Label-reference disagreement | 197 / 1446 (13.6%) |
| `row_ok=False` records | 65 (4.3%) |

### `Ten_example_Annotation.xlsx`

100 **real NHS epilepsy clinic letters** (from the King's College Hospital dataset used in
Fang et al. and Holgate et al.) annotated with the full multi-category binary schema used in those
studies. This is a separate dataset from the synthetic letters and is not directly comparable to
the JSON corpus. Its `seizure_frequency_number` column contains free-text annotations using the
Holgate et al. measurement system, not the Gan et al. Label Scheme.

## Label Taxonomy (Gan et al. Label Scheme)

The `seizure_frequency_number[0]` gold labels follow a structured canonical format defined in
Section 2.6.1 of Gan et al. Every label takes one of these six surface forms:

| Pattern | Example | Meaning |
|---|---|---|
| `unknown` | `unknown` | Seizures mentioned; no interpretable frequency |
| `no seizure frequency reference` | `no seizure frequency reference` | No seizure mention at all |
| `seizure free for <v\|multiple> <month\|year>` | `seizure free for 6 month` | Duration-based freedom |
| `<v\|multiple> per <v\|multiple> <day\|week\|month\|year>` | `4 per 6 week` | Rate-based frequency |
| `<v\|multiple> cluster per <v\|multiple> <day\|week\|month\|year>, <v\|multiple> per cluster` | `2 cluster per month, 6 per cluster` | Cluster-based frequency |
| `unknown, <v\|multiple> per cluster` | `unknown, 3 to 4 per cluster` | Clusters occur but spacing unknown |

`<v|multiple>` is either a numeric value (or range `a to b`) or the keyword `multiple`, which maps
to a predefined numeric value (3) for downstream conversion to seizures per month.

All labels deterministically convert to the Holgate et al. "seizures per month" scale, which is
then evaluated under the Purist or Pragmatic methods.

### Purist method (8-category)

| Category | Seizures/month target | Purist boundary |
|---|---|---|
| < 1 per 6 months | 0.08 | 0 < x ≤ 0.16 |
| = 1 per 6 months | 0.17 | 0.16 < x ≤ 0.18 |
| > 1/6M, < 1/M | ~0.33 | 0.18 < x ≤ 0.99 |
| = 1 per month | 1 | 0.99 < x ≤ 1.1 |
| > 1/M, < 1/W | ~2 | 1.1 < x ≤ 3.9 |
| = 1 per week | 4 | 3.9 < x ≤ 4.1 |
| > 1/W, < 1/D | ~15 | 4.1 < x ≤ 29 |
| ≥ 1 per day | 30–999 | 29 < x ≤ 999 |
| Unknown | 1000 | 1000 |
| No seizure information | 0 | 0 |

### Pragmatic method (2-category)

- **Infrequent**: 0 < x ≤ 1.1 (up to ~1 per month)
- **Frequent**: 1.1 < x ≤ 999 (more than ~1 per month)
- Unknown and No seizure information remain separate

The pragmatic split was chosen empirically to balance class sizes, not on clinical grounds.

## Issues Found

### Issue 1 — 197 label-reference disagreements reflect genuine letter ambiguity (not gold failures)

**Background**: `reference[0]` is an independent cross-check derivation of the label from the
letter. It is not the gold label; `seizure_frequency_number[0]` is gold (it matched the
construction-time ground truth in Step 7). The 197 cases (13.6%) where the two differ document
**letters where a second reading produces a different frequency interpretation**.

This is a feature, not a data quality failure — the letters are synthetically generated to cover a
wide range of frequency expressions, and some genuinely allow multiple valid readings. However,
the disagreements are informative for downstream use:

| Disagreement type | Count | Interpretation |
|---|---|---|
| Year/month precision | 26 | Secondary pass used coarser bucket (`multiple month` vs `1 year`) |
| Specific vs. unknown | 119 | Secondary pass didn't locate the frequency statement the primary pass found |
| Quantitative difference | 52 | Two readings of the same ambiguous text (range, cluster structure, etc.) |

The 119 specific-vs-unknown cases are the most significant: they suggest letters where the
frequency evidence is present but not obvious. These are high-value training examples (the primary
label is the correct one; the secondary pass's failure to find the evidence marks these as hard
cases). They can be used to create a `hard_cases` flag for stratified evaluation.

**Status**: No change needed to gold labels. Disagreement count can be used as a difficulty flag.

---

### Issue 2 — Label vocabulary has 403 unique values with surface-form fragmentation

**Observed**: 220/403 unique label values appear exactly once. The fragmentation is structural, not
erroneous — the Label Scheme generates combinatorially many valid surface forms. However, three
normalization issues exist that are not covered by the scheme's deterministic conversion rules:

**a) Plural inconsistency (true bug)**

`'seizure free for 6 month'` (n=14) and `'seizure free for 6 months'` (n=2) are
semantically identical but surface-distinct. Similarly `'seizure free for 2 year'` vs
`'seizure free for 2 years'`. String-equality comparison would score these as mismatches.

The Gan et al. label scheme specifies `<month|year>` in singular form. The plural variants are
malformed labels. **Fix**: strip trailing `s` from the time unit during normalisation, or reject
plural forms at labelling time.

**b) Year-to-month equivalence (scoring policy gap)**

`'seizure free for 1 year'` (n=9) and `'seizure free for 12 month'` (n=8) express the same
duration. The conversion rules should map both to the same seizures/month value. This is not
explicitly documented in the Gan et al. normalisation rules but is required for correct scoring.

The conversion: `seizure free for N year` → `seizure free for 12N month`, then the numeric
duration feeds into the seizure-freedom duration band but does not affect the frequency value
itself (seizure-free = 0 seizures/month = category NS in Holgate et al.).

**c) Cluster taxonomy with 96+ unique forms (expected but needs normalisation function)**

Cluster labels like `'1 cluster per week, 4 per cluster'` must be converted to seizures/month
by: (clusters/month) × (seizures/cluster). This requires unit normalisation for `per day/week/
year` → `per month` factors. The paper's `multiple → 3` rule applies to each slot independently.

**Status**: Items (a) and (b) require a normalisation pass before evaluation. Item (c) requires a
conversion function implementing the multiplication rule.

### Issue 3 — `reference[1]` contains three distinct evidence types

**Observed**: `reference[1]` stores:

| Type | Count | Example |
|---|---|---|
| Verbatim (or near-verbatim) clinical quote | ~1090 | `'She has had four focal seizures in the last three months'` |
| Short annotation context note (paraphrase) | ~356 | `'Startle-induced seizures'`, `'No seizures reported since last evaluation'` |
| LLM generation prompt (admin letters) | 54 | `'Create a reasonable NHS letter confirming cancellation…'` |

The 356 annotation notes are from the cross-check pass using a different evidence strategy —
the secondary annotator summarised the frequency context rather than quoting verbatim. This
explains the structure: both `seizure_frequency_number[1]` (evidence from primary pass) and
`reference[1]` (evidence from cross-check pass) can be valid, just differently styled.

For admin letters, `reference[1]` stores the generation prompt because these letters were
generated from prompts rather than from seizure-frequency descriptions, so there is no clinical
frequency evidence to quote.

**Impact on evaluation**: If `seizure_frequency_number[1]` is used as the ground-truth evidence
span for evidence-grounding evaluation, the 102 ellipsis-containing spans (see Issue 4) and the
54 null spans (admin letters) are the relevant cases. The `reference[1]` field should not be
used for evidence-grounding evaluation.

**Status**: Documented; no change to gold labels needed.

---

### Issue 4 — 102 evidence spans use `...` multi-span elision (7.1% of clinical records)

**Observed**: 102/1446 clinical records have `seizure_frequency_number[1]` containing `...`,
indicating the annotator bridged multiple non-adjacent letter passages into a single compound
evidence span.

**Why it matters**: A model trained or evaluated on verbatim quote matching would fail on these
spans. Gan et al. Format 4 specifies the evidence as text "copied from the input letter," which
implies verbatim extraction. The `...`-elided forms are not verbatim.

**Impact**: Evidence-grounding metrics (i.e., whether the model quotes the right passage) will be
unreliable for 7.1% of records unless multi-span evidence is explicitly handled by the scorer.
Frequency label scoring (which only uses `seizure_frequency_number[0]`) is unaffected.

**Status**: Flagged. Scorer needs to handle elided spans if evidence grounding is evaluated.

---

### Issue 5 — `row_ok=False` logic is correct but narrowly scoped

**Observed**: `row_ok=False` for exactly 65 records — the 54 admin letters plus 11 records with
`quote_issue_categories="Seizure Frequency Number"`. The 197 label-reference disagreements all
have `row_ok=True`.

**This is correct behavior**: the 54 admin letters are correctly labelled `'no seizure frequency
reference'` (= `NS` in Table 1, x=0) and excluded by `row_ok=False`. They are valid abstention
test cases (model should output `no seizure frequency reference` rather than hallucinate a
frequency). The 11 `quote_issue` records flag cases where the evidence span was flagged as
problematic by the cross-check pass.

**What `row_ok` does NOT do**: it does not flag the 197 ambiguous-reading records. This is
intentional — those records have valid gold labels that passed the Step 7 filter.

**Status**: Behavior is correct. If downstream consumers want to exclude hard cases, filter on
`seizure_frequency_number[0] != reference[0]` rather than on `row_ok`.

## Scoring Design

### Converting labels to x per month

The gold label string must be converted to a numeric seizures/month value before applying the
Purist or Pragmatic evaluation boundaries from Table 1. Conversion rules (from Gan et al.):

1. `no seizure frequency reference` → x = 0 (NS)
2. `seizure free for N <unit>` → x = 0 (NS) — the duration is informative for seizure-freedom
   analysis but the frequency is 0 seizures/month
3. `unknown` or `unknown, N per cluster` → x = 1000 (UNK)
4. `N per M <unit>` → x = (N / M) × unit_factor, where:
   - day → ×30, week → ×4.33, month → ×1, year → ÷12
   - `multiple` → 3
   - Ranges `a to b` → midpoint (a+b)/2
5. Cluster labels: x = (cluster_rate_per_month) × (seizures_per_cluster), where both factors
   follow the same `N per M <unit>` and `multiple` rules

### Plural normalisation (must apply before conversion)

Strip trailing `s` from time unit tokens: `months` → `month`, `years` → `year`, `weeks` → `week`,
`days` → `day`. This catches the malformed plural variants (Issue 2a).

### Evaluation methods

- **Purist**: 8 fine-grained temporal categories with fuzzy boundaries (soft logic)
- **Pragmatic**: 2 categories (infrequent ≤1.1/month vs frequent >1.1/month) + UNK + NS

Gan et al. report micro-F1 as the primary metric. UNK and NS are always scored as separate
categories. Seizure-freedom duration is not directly evaluated by either method (both map
seizure-free to NS/0).

## Relationship Between JSON and Excel

The Excel file contains **real** KCH NHS letters annotated with the 9-category ordinal frequency
schema from Holgate et al. The JSON contains **synthetic** letters annotated with the Gan et al.
Label Scheme. The two files use:

- Different letter populations (real vs. synthetic)
- Different annotation granularity (ordinal bucket vs. Label Scheme string)
- Different evidence formats (none in Excel vs. span-grounded in JSON)

For cross-corpus evaluation, synthetic Label Scheme labels convert to x/month (Gan et al. rules)
and real letters convert from annotator category to x/month (Holgate et al. Table 1 midpoints).
Both then evaluate under the same Purist/Pragmatic boundaries. This joint pipeline is described
in Gan et al. Section 2.9 and Tables 2–3.

## What Needs to Be Implemented

The following are gaps between the papers' description and what exists in the codebase:

1. **Label normalisation function** — converts `seizure_frequency_number[0]` strings to x/month.
   Must handle: plural stripping, `multiple→3`, range midpoints, unit factors, cluster
   multiplication, `unknown→1000`, `no seizure frequency reference→0`, `seizure free→0`.

2. **Purist and Pragmatic category assignment** — maps x/month values to the 8-class and 4-class
   schemas in Table 1.

3. **Hard-case flag** — mark the 197 records where `seizure_frequency_number[0] != reference[0]`
   for stratified reporting (to separately report performance on ambiguous letters).

4. **Multi-span evidence scorer** — if evidence grounding is evaluated, the scorer must handle
   `...`-elided spans (e.g., prefix match or first-span match rather than full-string match).

5. **Admin letter handling decision** — decide whether the 54 `row_ok=False` records should be
   included as abstention test cases or excluded entirely.
