## Changes Proposed Or Made

**No source changes made** (design lane only).

### Mission brief: isolated prompt-policy factor

**Varied factor (single):** *Qualitative frequency label evidence-gating* — tighten prompt policy so `infrequent`, `frequency same`, `frequency increased`, and `frequency decreased` are emitted only when the note contains independent lexical support, not because the label appears in the injected candidate list or from weak clinical inference.

**Rationale from A1 (largest precision leak, scorer-safe):**

| A1 signal | Count / detail |
| --- | --- |
| Qualitative over-emission category | 17 documents |
| FP label concentration | `infrequent` 12, `frequency same` 8, `frequency increased` 4, `frequency decreased` 1 |
| Evidence mismatch / candidate echo | 12 documents (e.g. EA0174 cites injected candidate prompt) |
| Rejected alternative | High-precision candidate pruning (−12.0pp recall cap-25, −4.2pp F1) — candidate density must stay high-recall |

**Explicitly out of scope for this card (deferred):**

- Temporal/current-scope rules (historical rates, limited seizure-free windows) — separate factor; overlaps A2 verifier territory
- Multi-type/range normalization policy — separate factor
- Gold-empty clinical-frequency documents — benchmark caveats only; do not encode as prompt “clinical truth”
- Candidate list density, scorer, bridges, AM guard primitive — frozen controls

### Hypothesis

Tightening qualitative evidence requirements in prompt guidance (without shrinking the pre-vocab candidate list) will reduce precision-dominated false positives (`infrequent`, `frequency same`, candidate echo) while preserving the 86.0% full-validation recall that high-recall candidates enable.

**Prompt guidance additions** (to `EXECT_S4_FREQUENCY_LABEL_POLICY_GUIDANCE` only):

1. **Independent note support required for qualitative labels.** Emit `infrequent`, `frequency same`, `frequency increased`, or `frequency decreased` only when the note contains explicit supporting wording (e.g. “infrequent”, “same frequency”, “increased/decreased frequency”, “more/less frequent”). Precomputed candidate hints alone are insufficient.
2. **No candidate-list echo.** Do not emit a qualitative label whose sole rationale is presence in the injected candidate line; evidence quotes must come from the clinical note body below the `---` separator.
3. **No cross-field inference.** Medication control, rescue-medication use, or seizure-type descriptions do not justify qualitative frequency labels unless frequency change is explicitly stated (targets EA0029-style FPs).
4. **Preserve quantified + supported qualitative co-emission.** When the note explicitly states both a quantified rate and a frequency change, emit both (existing v1_2 behavior retained).
5. **Abstain over weak qualitative inference.** Prefer empty `seizure_frequency` over emitting unsupported qualitative labels; do not treat gold-empty notes as license to invent frequency labels.

**Candidate injection header** — optional one-line clarification in `format_exect_frequency_pre_vocab_note()` (prompt-surface only, same high-recall builder):

> “Candidates are soft hints; qualitative labels still require independent note support.”

This does **not** change `_build_exect_frequency_label_set()` output or candidate count.

**Policy examples to add** (negative + recall-preserving positive), mapped from A1:

| Case ID | A1 doc | Type | Intent |
| --- | --- | --- | --- |
| `s4_qual_no_infrequent_without_wording` | EA0008 | Negative | Gold `1 per 3 week` + `frequency increased`; do not add `infrequent` without note wording |
| `s4_qual_med_control_not_freq_decreased` | EA0029 | Negative | Medication-control language ≠ `frequency decreased` |
| `s4_qual_no_candidate_echo` | EA0174 | Negative | Do not emit rate/qualitative label citing candidate prompt; require note quote |
| `s4_qual_infrequent_when_stated` | EA0059 | Positive (recall guard) | Emit `infrequent` when note explicitly uses infrequent/rare wording |
| `s4_qual_seizure_free_candidate_present` | EA0173 | Positive (recall guard) | Emit `seizure free` when note supports it even if model hesitates — **seizure free is not in the isolated factor** but example prevents over-abstention bleed |

Note: `seizure free` / zero-rate / temporal-scope rules are **not** varied in this arm; examples mentioning `seizure free` serve only as guardrails against collateral recall loss. Primary metric target remains qualitative FP reduction.

## Risks

1. **Over-abstention on true qualitative labels.** A1 has 6 FNs including `infrequent` (EA0059) and `seizure free` (EA0173, EA0137, EA0143). Aggressive evidence-gating could trade precision for recall loss; cap-25 recall floor is mandatory.
2. **Prompt-only ceiling.** A1 shows 10 gold-empty clinical-frequency FPs and 9 gold-policy ambiguity cases that prompt policy cannot fix without scorer/gold changes; expect residual FPs even if qualitative gating works.
3. **Candidate echo vs. legitimate co-occurrence.** Notes that genuinely support `infrequent` may also appear in candidate lists; policy must distinguish support source, not ban labels merely because they were hinted.
4. **Cross-family prompt length drift.** High-precision cap-25 showed seizure_type regression (−4.3pp) from context changes; monitor guard families on cap-25 even though A3 edits target frequency guidance only.
5. **A2/A3 file collision.** If A2I verifier pilot touches `exect_s4.py` or shared prompt modules concurrently, implementations must stay single-threaded per campaign plan.
6. **Missing local run artifacts.** Cap-25/full AM-guard reruns required in A3I before any promotion claim; this brief cannot pre-certify metric deltas.
7. **Temporal/multi-type residuals untouched.** ~13 documents (temporal + multi-type categories) will remain after qualitative gating; do not interpret partial FP reduction as mechanism closure.
