Stress-test my plan, but ground every question in the repository.

Before asking me anything:
1. Read AGENTS.md and relevant local docs.
2. Search the codebase for existing patterns.
3. Identify likely affected modules, tests, schemas, APIs, and config.
4. Answer any question that can be answered from the repo yourself.

Then interview me one question at a time.

For each question:
- cite the repo evidence or file path that motivated it
- explain the decision it blocks
- provide your recommended answer
- wait for my answer before continuing
